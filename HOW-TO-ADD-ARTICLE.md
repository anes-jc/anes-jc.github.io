# 記事を1本ふやす手順（コード不要）

> 毎週日曜日の「先週のまとめ｜最新論文3選」は、校正・承認不要の全自動記事として生成・公開されます。設定と運用は `SUNDAY-ARTICLE-AUTOMATION.md` を参照してください。

記事の一覧は **`data/articles.js` という1つのファイル**で管理しています。
記事を足すときに編集するのは、基本この1ファイルだけです。

---

## 全体の流れ（3ステップ）

```
① 記事ファイルを articles/ に置く
        ↓
② data/articles.js のリストに1ブロック足す
        ↓
③ （ふだんは不要）裏方の台帳 data/history.json を更新
```

トップページの一覧・スマホメニューの新着・タグページは、
②のリストを読んで**公開日になった記事だけ自動で並びます**。HTMLを貼る作業はもうありません。

---

## ① 記事ファイルを置く

Claude と一緒に作った記事HTML（例 `xxx.html`）を、
GitHub の `articles/` フォルダにアップロードします。

- GitHub のリポジトリ → `articles` フォルダを開く
- **Add file → Upload files** で記事HTMLをドラッグ
- **Commit changes**

---

## ② data/articles.js に1ブロック足す

GitHub で `data/articles.js` を開き、鉛筆（Edit）。
`window.ARTICLES = [` のすぐ下に、新しいブロックを**1つ足す**だけです。
一番上に足すと、トップページでも一番上（最新）に並びます。

既存の1件をコピーして、中身を書き換えるのが一番安全です。形はこれです：

```javascript
  {
    date: "2026.06.22", dow: "MON", status: "公開予定", live: false,
    title: "記事のタイトル",
    url: "articles/xxx.html",
    desc: "一覧に出る短い説明文。",
    dowTag: { label: "MON · 新着RCT", tag: "新着RCT" },
    tags: [
      { label: "麻酔", tag: "麻酔" },
      { label: "気道管理", tag: "気道管理" },
      { label: "G01 研究デザインを読む", tag: "研究デザインを読む" }
    ]
  },
```

### 各項目の意味
| 項目 | 何を書くか |
|------|-----------|
| `date` | 表示する日付。例 `"2026.06.22"` |
| `dow`  | 曜日。`"MON"` / `"WED"` / `"FRI"` |
| `status` | 状態ラベル。未来の記事は `"公開予定"`、公開済みは `"公開中"` |
| `live` | 未来の記事は `false`、公開済みは `true`。表示時には `date` から日本時間で自動判定される |
| `title` | 記事タイトル |
| `url` | 記事ファイルの場所。`"articles/ファイル名.html"` |
| `desc` | 一覧に出る短い説明 |
| `dowTag` | 曜日タグ。`label`が表示文字、`tag`がリンク先 |
| `tags` | タグの配列。`label`が表示文字、`tag`がリンク先 |

### 書くときの注意（これだけ守ればOK）
- **各行のおわりのカンマ `,` を消さない**（ズレると表示されなくなります）
- ブロックのおわりの `},` も忘れずに
- 公開枠は **月曜＝新着RCT、 水曜＝統計・手法、 金曜＝古典論文**。`date` の実際の曜日、`dow`、`dowTag` を必ず一致させる
- 同じ曜日枠の記事候補が複数ある場合は、同日に重ねず、次の週の同じ曜日へ送る
- **統計テーマのタグ**は、`label` を `"G02 効果の大きさを読む"` のように
  グループ番号つきで書く（番号は自動で薄く表示されます）。
  `tag` のほうは番号なしの `"効果の大きさを読む"` にする
- 迷ったら、既存のブロックを丸ごとコピーして書き換える

書き換えたら **Commit changes**。数分でサイトに反映されます。

---

## ③ 裏方の台帳（ふだんは気にしなくてOK）

`data/history.json` は「同じ論文を二度使わない」ためのチェック台帳です。公開済みだけでなく、公開予定の記事も含めます。
自動化を入れるまでは、Claude が更新版を渡すので**差し替えるだけ**。
手で触らなくても、サイトの表示には影響しません。

---

## よくあるトラブル

| 症状 | 原因と対処 |
|------|-----------|
| 足した記事が一覧に出ない | `data/articles.js` のカンマ・`{ }` のズレ。直前のブロックと見比べる |
| 記事を開くと404 | `url` のファイル名と、`articles/` に置いたファイル名が違う |
| タグを押しても0件 | `tag`（リンク先）の文字が他の記事と微妙に違う（表記ゆれ）。統一する |
| 反映されない | ブラウザを強制リロード（Ctrl+Shift+R / Cmd+Shift+R） |

## なぜこの作りなのか（おぼえなくていい豆知識）

記事情報を1か所（`articles.js`）にまとめておくと、トップ・スマホメニュー・
タグページが全部そこを読んで自動で並びます。だから記事を足すときに
複数のページを直す必要がなく、間違いも起きにくい。
将来 Claude API での自動化に進むときも、この `articles.js` に
1ブロック自動で書き足すだけ、という形にできます。

## Xリンクカード

- 記事HTMLには og:* と twitter:* メタタグを設定する。
- 通常記事のカード画像は assets/og/<記事slug>.png に置く。
- 日曜記事は assets/og/sunday-x-header.png を共通利用する。
- URLは https://anes-jc.github.io/ から始まる絶対URLにする。

## 検索向けメタ情報

- 記事HTMLの `<head>` には、記事ごとの `description`、`canonical`、`og:*`、`twitter:*` を入れる。
- `description` は汎用文ではなく、論文名・対象・統計テーマがわかる1文にする。
- `script type="application/ld+json"` で `BlogPosting` を入れる。
- `BlogPosting` には `headline`、`description`、`image`、`datePublished`、`dateModified`、`author`、`publisher` を入れる。
- 公開済みURLの sitemap は `node scripts/generate-sitemap.mjs` で生成する。
