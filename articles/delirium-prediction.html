ICUの譫妄を予測できるか — 機械学習モデルと ROC・AUC の読み方

A retrospective study based on MIMIC-IV.

01

臨床背景：なぜこの研究が必要だったか

譫妄はICUで最も多い急性脳機能障害で、予後悪化・在院延長・コスト増につながります。とりわけCOPDと呼吸不全を併せ持つ高齢者は高リスクですが、この集団に特化した早期予測ツールはほとんどありませんでした。

「誰が譫妄を起こしそうか」を入室時点で見積もれれば、予防的ケアを先回りで集中投下できます。そこで大規模公開データベース（MIMIC-IV）を使い、機械学習で予測モデルを作ったのが本研究です。臨床の問いと同時に、予測モデルをどう評価するかという統計の作法を学ぶのに格好の題材です。

03

研究デザイン解説

これはRCTではなく予測モデル開発研究です。読むときの軸がRCTと違います。RCTが「介入の効果」を問うのに対し、予測モデルは「アウトカムをどれだけ当てられるか」を問います。本研究は単一データベースを80:20で訓練用と検証用に分け、訓練過程で10分割交差検証を行いました。欠測率10%超の変数を除外し、残る欠測値をランダムフォレストによる多重代入で補完し、クラス不均衡にはSMOTEを使用しています。

注意したいのは、これが外部検証ではなく内部検証である点。同じMIMIC-IVを分けて使っているので、別の病院・別の集団でも同じ性能が出るかは別問題です。予測モデル論文では「どこまでが内部で、どこからが外部か」を必ず確認します。

04

今回の統計キーポイント

STATS KEY · 統合・予測・診断を読む

ROC曲線とAUC、そして「識別」と「較正」は別物

予測モデルの性能は、まず ROC曲線で見ます。これは閾値を動かしたときの「感度（本当に譫妄の人を当てる率）」と「1−特異度（譫妄でない人を誤って陽性とする率）」の関係を描いた曲線です。その曲線下の面積が AUC。

AUC = 0.5 → でたらめ（コイン投げと同じ）
AUC = 0.7–0.8 → まずまず
AUC = 0.8–0.9 → 良好
AUC = 0.932 → 非常に高い識別能（本研究のXGBoost）

AUCの直感的な意味は「譫妄になった人とならなかった人を1人ずつ取り出したとき、モデルが前者により高いリスクを付ける確率」。0.932なら約93%の確率で正しく順序づけられる、という識別能の高さを表します。

ただしAUCが高い＝良いモデル、ではありません。AUCは識別（discrimination：順位づけの能力）しか見ていません。もう一つ較正（calibration：予測した確率が実際の発生率と一致するか）という軸があります。「80%と予測した人の実際に80%が譫妄になるか」が較正です。

識別（AUC）… 高リスクと低リスクを区別できるか
較正（calibration）… 予測確率そのものが当たっているか
→ 両方そろって初めて「使える」モデル

本研究は決定曲線分析（DCA）を示していますが、本文・図には較正曲線や較正指標の報告を確認できませんでした。したがって「予測確率そのものが正しい」とは評価できません。予測モデル論文を読むときは「AUCだけでなく較正とDCAを見たか？」が定型チェックです。

05

主な結果

対象は1,155例。譫妄数は抄録では149例（12.9%）、本文Resultsでは146例と原著内で不一致があります。4モデルのうち XGBoost が最良の識別能を示し、検証セットでAUC 0.932、正確度0.891、適合率0.839、再現率0.795、F1スコア0.810でした。DCAは介入に有利な結果と著者らが報告していますが、較正は報告されていません。

SHAP解析で重要だった予測因子は、GCS言語スコア・在院日数・ICU初日の平均SpO₂・腎機能（MDRD）・平均拡張期血圧・GCS運動スコア・性別・非侵襲換気の時間など。意識レベルと酸素化が上位に来るのは臨床感覚とも合致します。

06

リミテーション

  • 単一データベース（MIMIC-IV）の内部検証のみ。外部の別集団での性能は未確認で、過学習の可能性が残る。
  • 後ろ向きデータで、譫妄はICD-9/10診断コードにより定義される。未診断・未記録例を見落とす可能性がある。
  • 譫妄数が抄録149例、本文146例と一致しない。イベント数と性能指標の再現確認が必要。
  • AUC 0.932は検証セット、0.921は10分割交差検証で報告された値。評価条件を区別して読む必要がある。
  • 予測因子に在院日数や非侵襲換気時間が含まれるため、「早期」に利用できる時点より後の情報が混入する時間的情報漏洩の懸念がある。
  • 較正曲線・較正指標が報告されておらず、個々の予測確率の正確さは判断できない。
07

結果をどう解釈するか

高いAUCは魅力的ですが、予測モデルの価値は「利用時点で入手可能な情報だけを使い、外部集団でも、確率が当たるか」で決まります。本研究は内部検証で優秀な識別能を示した第一歩ですが、較正は未報告で、在院日数などの時間的情報漏洩も懸念されます。明確な予測時点を定めた別施設での外部検証を経て初めて臨床ツールとして検討できます。

読み手としての教訓は、「AUC 0.9超え」という見出しに飛びつかず、較正・DCA・検証の種類（内部か外部か）まで確認する習慣です。識別だけを見て「当たるモデル」と判断するのは、複合アウトカムで内訳を見ないのと同じ落とし穴です。

08

現代の視点

現在の臨床予測モデル評価では、単一施設データのランダム分割で高いAUCを示すだけでは十分ではありません。対象患者をいつ識別し、どの時点までの変数で、いつ以後の譫妄を予測するのかを事前に固定する必要があります。外部・時間的検証、較正、欠測処理、クラス不均衡処理、臨床導入後の影響評価まで確認して、初めて実装可能性を論じられます。

本モデルはコードが公開されている点は検証可能性を高めます。一方、在院日数を含むモデルを「早期予測」に使うには再設計が必要です。現時点ではベッドサイドの意思決定ツールではなく、将来の外部検証に値する開発モデルとして位置づけるのが妥当です。

09

英語で読む

予測モデル論文に頻出する評価指標の英語を確認します。診断・予測の語彙はパターンが決まっているので、一度覚えると応用が利きます。

— Results, Abstract（PLOS One 2025;20:e0319297）より一文を引用 "The XGBoost model demonstrated the best discriminative ability, achieving an AUC of 0.932 in the validation set."

読みのポイント：discriminative ability（識別能）は高低リスクを区別する力で、AUCで測ります。demonstrated は結果報告の定番動詞。validation set は訓練に直接使わず性能評価に使うデータ集合で、ここでは同じMIMIC-IV内の分割のため外部検証ではありません。予測モデルでは discrimination と calibration を分け、どのデータで検証したかまで確認します。

10

明日からの臨床に活かすには

TAKEAWAY

「AUC 0.9超えの譫妄予測モデル」という話を聞いたら、まず予測時点・較正・外部検証を確認する。識別能（AUC）が高くても、未来の情報が混入していたり、予測確率が未評価だったり、別集団で性能が落ちたりすれば、ベッドサイドでは使えません。本モデルを直接利用するのではなく、譫妄リスク評価と標準的な予防・モニタリングを丁寧に行う契機として読みます。
