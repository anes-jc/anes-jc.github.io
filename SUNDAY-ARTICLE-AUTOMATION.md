# 日曜「先週のまとめ」全自動公開

## 運用ルール

- 毎週日曜日 18:00 JSTに通常記事として自動生成・自動公開
- 19:17 JSTに補完実行
- ダブルチェック、独立校正、人間の最終確認は不要
- 新規記事の人間確認ルールの例外として扱う
- 各日曜記事の本文内に、自動生成・自動公開の週次まとめであることを明記する
- 月・水・金の記事作成フローには影響しない

## 自動生成される記事

- タイトル: `先週のまとめ｜最新論文3選`
- URL: `articles/latest-papers-YYYY-MM-DD.html`
- 先週公開したサイト記事: タイトル・公開日・リンクのみ
- 最新論文3選: 日本語の注目テーマを大見出しで表示
- 最新論文の大見出しは、領域名だけでなく「対象・比較または評価内容・主要アウトカム」が分かる研究質問型にする
- 自動生成時は論文タイトル・抄録の主要キーワードから具体的な日本語見出しを選び、該当ルールがない場合は「注目テーマ：研究デザイン」の形式にする
- 週次ページのレイアウトは通常記事と同じ固定ヘッダー、記事ヘッド（kicker・引用情報欄）、番号付きセクション、本文外フッターを使用する
- 冒頭セクションに「この日曜まとめについて」を置き、自動生成・自動公開であること、月水金の個別記事は人間確認後に公開されることを説明する
- 各論文には研究デザイン、英語タイトル、雑誌名、原文リンクを表示
- 研究プロトコル、editorial、letter は最新論文3選から除外

英語抄録全文やRSS専用ページは生成しません。

## 全自動公開

GitHub Actionsの `Publish Sunday article` が以下を実行します。

1. Europe PMCから対象週の最新論文を取得
2. 注目テーマが異なる論文を優先して3件選定
3. 通常記事HTMLを生成
4. `data/sunday-articles.js` へ記事情報を追加
5. `sitemap.xml` を更新
6. GitHubへ自動コミット・公開

Europe PMCが一時的に応答しない場合は3回再試行します。論文3件を取得できない場合は不完全な記事を公開せず失敗扱いとし、19:17 JSTの補完実行へ回します。同じ日に再実行しても同じ記事を更新するため、重複記事は作成されません。

## 設定

検索条件・優先ジャーナル・免責文は `data/sunday-article.json` で変更できます。

ローカル確認:

```powershell
$env:ISSUE_DATE_JST = "2026-06-14"
node scripts/generate-sunday-article.mjs
```

## 互換性

- 月・水・金の記事追加方法と既存記事スキーマは変更しません。
- 日曜記事は `data/sunday-articles.js` で別管理し、トップページとタグ一覧で通常記事として統合表示します。

- Xリンクカードには assets/og/sunday-x-header.png を使用し、生成時にメタタグを自動設定します。
- 検索向けに `BlogPosting` のJSON-LDを自動設定し、`scripts/generate-sitemap.mjs` で公開済みURLの sitemap を更新します。
