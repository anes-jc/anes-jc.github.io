# 公開手順 — このフォルダをホームページにする

このフォルダ（`site`）の中身を GitHub に置くと、無料でホームページとして公開できます。プログラミングは不要です。画面の通りに進めれば 15〜20 分で公開できます。

> 自動更新（論文収集→AI下書き→レビュー→公開）の仕組みは、この公開が終わってから次のステップで足します。**まずは「サイトをインターネットに出す」ことだけ**やります。

---

## このフォルダの中身

```
site/
├── index.html              ← トップページ（統計テーマのタグつき）
├── tags.html               ← テーマ別の記事一覧（タグをクリックすると開く）
├── articles/
│   ├── nice-sugar.html             ← 公開中の記事
│   └── その他の記事.html           ← 公開済み・公開予定の記事
├── data/
│   ├── articles.js                ← 記事一覧と公開予定日（公開日になると自動表示）
│   ├── classics.json              ← 金曜用の古典論文リスト
│   ├── paper-selection-policy.json ← 月・水・金の論文選定ルール
│   ├── stat-themes.json           ← 統計テーマの台帳
│   └── history.json               ← 公開済み・公開予定を含む重複防止台帳
├── .nojekyll               ← GitHub Pages 用の設定（中身は空。消さないでください）
└── README.md               ← このファイル
```

---

## ステップ 1 — GitHub アカウントを作る（持っていれば飛ばす）

1. <https://github.com/signup> にアクセス
2. メールアドレス・パスワード・ユーザー名を登録（ユーザー名は公開URLに使われます。例：`taro-anes`）
3. 無料プランでOK

---

## ステップ 2 — リポジトリ（置き場所）を作る

1. ログイン後、右上の「＋」→ **New repository**
2. **Repository name** に `anesthesia-study` と入力（好きな名前でOK）
3. **Public** を選択（ホームページとして公開するため）
4. ほかは空のまま **Create repository** をクリック

---

## ステップ 3 — ファイルをアップロードする

1. できたリポジトリの画面で **uploading an existing file** というリンクをクリック
   （または **Add file → Upload files**）
2. この `site` フォルダの**中身**（`index.html`, `articles`, `data`, `.nojekyll`, `README.md`）を
   まとめてドラッグ＆ドロップ
   - ⚠️ `site` フォルダごとではなく、**中身**を入れてください
   - ⚠️ `.nojekyll` が見えない場合：Mac は `Command+Shift+.`、Windows はエクスプローラーの「表示→隠しファイル」で表示できます。これも必ず入れてください
3. 下の **Commit changes**（緑のボタン）をクリック

---

## ステップ 4 — ホームページ機能をオンにする（GitHub Pages）

1. リポジトリ上部の **Settings**（歯車）をクリック
2. 左メニューの **Pages** をクリック
3. **Source** で **Deploy from a branch** を選ぶ
4. **Branch** で `main` を選び、フォルダは `/ (root)` のまま **Save**
5. 1〜2分待つと、同じ画面の上部に
   `Your site is live at https://（あなたのユーザー名）.github.io/anesthesia-study/`
   と表示されます。これが**あなたのホームページのURL**です

スマホでもPCでも、このURLを開けばサイトが見られます。✅ ここまでで公開完了です。

---

## うまく表示されないとき

| 症状 | 対処 |
|------|------|
| 404 と出る | 公開直後は反映に数分かかります。少し待って再読み込み |
| 記事リンクを押すと開けない | `articles` フォルダがアップロードされているか確認 |
| レイアウトが崩れる | `.nojekyll` がアップロードされているか確認（これが無いと崩れます） |
| 変更が反映されない | ブラウザの再読み込み（強制リロード：`Ctrl+Shift+R` / `Cmd+Shift+R`） |

---

## 記事を直したい・差し替えたいとき（手動の場合）

GitHub のリポジトリで対象ファイルを開き、鉛筆アイコン（Edit）→ 編集 → **Commit changes**。
数分でサイトに反映されます。

---

## 独自ドメインを使いたいとき（任意・あとででOK）

`〜.github.io/...` のURLのままでも問題なく運用できます。独自ドメイン（例：`anes-study.jp`）を使いたくなったら、ドメインを取得して Settings → Pages の **Custom domain** に入力すれば設定できます。手順はそのとき改めて案内します。

---

## 次のステップ（公開できたら教えてください）

公開URLが出たら、次は自動更新の仕組みを足します。順番はこうです：

1. **テンプレート化** … サンプル記事を、AIが穴埋めできる「型」に変換
2. **論文収集スクリプト** … 対象誌のOA新着＋金曜古典を自動で集める
3. **AI下書きスクリプト** … Claude API で記事を生成（要：APIキー登録）
4. **自動実行＆レビューの設定** … 月水金に自動で下書き→あなたが承認して公開

このとき、あなたにお願いするのは「APIキーの登録」と「レビュー（承認ボタン）」くらいです。各場面で具体的に指示します。
